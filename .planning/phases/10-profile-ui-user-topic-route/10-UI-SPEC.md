## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Touch targets use `min-h-[44px]` (44px, not a multiple of 4 — matches existing NavBar and ProfilePage button pattern).

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-gray-50` (#F9FAFB) | Page background, surfaces |
| Secondary (30%) | `bg-white` (#FFFFFF) | Cards, nav bar, form containers |
| Accent (10%) | `#4F46E5` (indigo-600) | Primary buttons, active nav links, focus rings |
| Destructive | `bg-red-600` / `text-red-600` | Destructive actions, error messages |

Accent reserved for: Primary CTA buttons (`bg-primary`), active navigation link underlines (`border-primary`), and focus ring outlines (`ring-primary-ring`).

## Layout Contract

### Profile Page Sections (top to bottom)

```
┌─────────────────────────────────────────────────┐
│ My Profile (h2, text-2xl font-bold)             │
├─────────────────────────────────────────────────┤
│ [Card] User Info                                │
│   Avatar circle + name + email + role            │
├─────────────────────────────────────────────────┤
│ [Card] Change Password                          │
│   Current password → New password → Confirm      │
│   "Update Password" button                      │
├─────────────────────────────────────────────────┤
│ [Card] Display Color                            │
│   Color picker + "Update Color" button           │
├─────────────────────────────────────────────────┤
│ [Card] Push Notifications ← NEW IN PHASE 10     │
│   Section title: "Push Notifications"           │
│   Help text: "Set your ntfy topic..."           │
│   ┌─ View Mode ────────────────────────────┐    │
│   │ Current topic displayed, or empty state │    │
│   │ "Change" button to enter edit mode      │    │
│   └─────────────────────────────────────────┘    │
│   ┌─ Edit Mode ────────────────────────────┐    │
│   │ Topic input (text, monospace)           │    │
│   │ "Generate random topic" button          │    │
│   │ "Save" + "Cancel" buttons               │    │
│   └─────────────────────────────────────────┘    │
│                                                  │
│   ┌─ Family Topics (PARENT only) ──────────┐    │
│   │ Heading: "Family Topics"                │    │
│   │ Card per family member (excluding self) │    │
│   │   Name + topic (or "Not set") + Edit    │    │
│   │   Edit mode: inline input + Save/Cancel │    │
│   │   Generate random topic available        │    │
│   └─────────────────────────────────────────┘    │
└─────────────────────────────────────────────────┘
```

## Responsive Behavior

| Viewport | Behavior |
|----------|----------|
| Desktop (≥768px) | Full-width cards, `max-w-4xl mx-auto` container |
| Mobile (<768px) | Same layout, `px-4` padding, full-width inputs, stacked buttons |

No additional responsive changes needed — the existing Profile page layout works on all viewports. Topic input is a single-line text field that naturally scales.
