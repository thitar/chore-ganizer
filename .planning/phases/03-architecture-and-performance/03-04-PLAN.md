### Migration approach (D-08)
1. Delete old migration files — they were created by `prisma db push` cycles and may not reflect actual schema state
2. Generate a fresh baseline using `prisma migrate diff` (non-interactive, works without DB):
   ```bash
   npx prisma migrate diff --from-empty --to-schema-datasource prisma/schema.prisma --script
   ```
   This produces a SQL CREATE TABLE script for all 16 models
3. Create migration directory: `prisma/migrations/0_baseline/migration.sql`
4. Update `prisma/migrations/migration_lock.toml` 
5. Update `docker-entrypoint.sh`: replace `npx prisma db push --skip-generate` with `npx prisma migrate deploy`

### Handling existing databases
Existing databases created via `db push` have NO `_prisma_migrations` table. For `prisma migrate deploy` to work:
- Before running migrate deploy, mark the baseline migration as applied:
  ```bash
  npx prisma migrate resolve --applied 0_baseline 2>/dev/null || true
  ```
- This tells Prisma "this migration has already been applied" so it doesn't try to create tables that already exist.
- Then `prisma migrate deploy` will only apply NEW migrations in the future.

### About the seed password warning (D-04)

- The warning should be prominent: `echo "WARNING: Default passwords in use — change immediately for production"`
- Logged after seeding completes (line ~126) so operators see it during startup
- Only logged when seeding actually runs (not on every restart)
