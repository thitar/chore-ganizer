<feature>
  <name>CSRF Token Retry Loop Prevention</name>
  <files>frontend/src/api/client.ts, frontend/src/api/client.test.ts</files>
  <behavior>
    - When a request receives 403 with CSRF_TOKEN_INVALID/CSRF_TOKEN_MISSING:
      - If no retry has been attempted for this request: fetch new CSRF token and retry exactly once
      - If a retry has already been attempted: propagate the error to the caller (do not retry again)
    - The retry state must be tracked per-request (not global) to allow unrelated requests to retry independently
    - Edge case: If CSRF token refresh itself fails, the retry should still only happen once
  </behavior>
  <implementation>
    Add a `maxRetries` or `retryCount` property to the request config (Axios supports custom config properties) or track via a WeakMap keyed by the request config object.
  </implementation>
</feature>
