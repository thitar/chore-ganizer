    1. In `.github/workflows/ci-cd.yml`, add a new job or step early in the workflow (before backend/frontend jobs) named "Validate Version Sync":

    ```yaml
    validate-versions:
      name: Validate Version Sync
      runs-on: ubuntu-latest
      steps:
        - name: Checkout code
          uses: actions/checkout@v6
        
        - name: Check backend/frontend version match
          run: |
            BACKEND_VERSION=$(grep '"version"' backend/package.json | head -1 | sed 's/.*"\([^"]*\)".*/\1/')
            FRONTEND_VERSION=$(grep '"version"' frontend/package.json | head -1 | sed 's/.*"\([^"]*\)".*/\1/')
            echo "Backend version: $BACKEND_VERSION"
            echo "Frontend version: $FRONTEND_VERSION"
            if [ "$BACKEND_VERSION" != "$FRONTEND_VERSION" ]; then
              echo "Error: Backend version ($BACKEND_VERSION) does not match frontend version ($FRONTEND_VERSION)"
              exit 1
            fi
            echo "Versions match: $BACKEND_VERSION"
    ```

    2. Add `needs: validate-versions` to both the `backend` and `frontend` jobs so they only run after version validation passes.

    Do NOT modify the existing backend/frontend job logic beyond adding the `needs` dependency.
  </action>

    # Sync .env APP_VERSION if it exists and differs

        if [ "$ENV_VERSION" != "$APP_VERSION" ]; then
            echo "Updating .env APP_VERSION from $ENV_VERSION to $APP_VERSION"
