<objective>
Harden the existing backend authentication against three security gaps identified in cross-AI review: session fixation (no `session.regenerate()` on login), ghost sessions (deleted users retain access), and cookie residue (logout relies on `session.destroy()` alone without explicit `clearCookie`).
Also cross-verify that `app.ts` session cookie attributes (httpOnly, sameSite) are correctly configured and note bcrypt salt alignment with the Phase 1 seeder.

Purpose: Close two HIGH-severity and two MEDIUM/LOW-severity security gaps before later phases build on the auth layer. Each fix is a small, targeted code change but carries disproportionate security value.
Output: Hardened auth routes and middleware with updated test coverage proving the fixes work.
</objective>
