    **Changes in useAuth.tsx:**

    1. **Change `staleTime` from `5 * 60 * 1000` to `Infinity`:**
       - In the `useQuery({ queryKey: ['auth', 'me'], ... })` options object (line 20–21), change:
         `staleTime: 5 * 60 * 1000,` → `staleTime: Infinity,`
       - Auth state only changes on explicit login/logout actions — window focus should not trigger revalidation. This prevents the flash-of-auth-check and unnecessary server calls.

    2. **Add `queryClient.clear()` in logout mutation onSuccess:**
       - In the `logoutMutation` `onSuccess` callback (currently `() => queryClient.setQueryData(['auth', 'me'], null)`), ADD `queryClient.clear()` BEFORE setting query data:
       ```
       onSuccess: () => {
         queryClient.clear()                        // ← NEW: wipe ALL cached queries
         queryClient.setQueryData(['auth', 'me'], null)
       }
       ```
       - `queryClient.clear()` removes every cached query result from React Query's cache. This prevents a parent who logs out from leaving chores, points, or user data in the cache that a child logging in next would inherit.

    3. **Update login mutation to use `invalidateQueries` instead of refetch:**
       - The existing `onSuccess: () => queryClient.invalidateQueries({ queryKey: ['auth'] })` is correct — it triggers a fresh GET /me after login. Keep as-is. With `staleTime: Infinity`, `invalidateQueries` is the ONLY mechanism that will refetch the auth query (which is the correct design).

    4. **No other files need changes for this task.** The scaffold test mocks auth.api.ts functions and doesn't test query cache behavior, so it doesn't need updating for this change.
