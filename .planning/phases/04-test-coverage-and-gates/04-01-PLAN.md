### Testing pattern for each controller test

```typescript
// Pattern: controller that delegates to services
import * as controllerModule from '../../controllers/{name}.controller'
import * as serviceModule from '../../services/{name}.service'

// Mock the service
jest.mock('../../services/{name}.service', () => ({
  functionName: jest.fn(),
}))

describe('{ControllerName}', () => {
  let mockReq: ReturnType<typeof createMockRequest>
  let mockRes: ReturnType<typeof createMockResponse>

  beforeEach(() => {
    jest.clearAllMocks()
    mockReq = createMockRequest({ user: { id: 1, role: 'PARENT' } })
    mockRes = createMockResponse()
  })

  describe('handlerName', () => {
    it('should return 200 with data on success', async () => {
      const mockData = { ... }
      ;(serviceModule.functionName as jest.Mock).mockResolvedValue(mockData)

      await controllerModule.handlerName(mockReq as any, mockRes as any)

      expect(mockRes.status).toHaveBeenCalledWith(200)
      expect(mockRes.json).toHaveBeenCalledWith({
        success: true,
        data: mockData,
      })
    })

    it('should propagate service errors', async () => {
      const error = new Error('Service error')
      ;(serviceModule.functionName as jest.Mock).mockRejectedValue(error)

      await expect(
        controllerModule.handlerName(mockReq as any, mockRes as any)
      ).rejects.toThrow('Service error')
    })
  })
})
```

### Common controller patterns to test

**Auth-gated routes** (most controllers):
- `req.user` is undefined → handler throws AppError('User not authenticated', 401, 'UNAUTHORIZED')
- `req.user.role !== 'PARENT'` → handler throws AppError (for parent-only routes)

**Param parsing validation** (chore-assignments, users, etc.):
- Missing required body field → throw AppError('X is required', 400)
- Invalid param (NaN userId from req.params) → throw AppError('Invalid user ID', 400)

**Unused-param handlers** (`_req: Request, res: Response`):
- healthCheck, getOverdue, getTemplates, livenessCheck, readinessCheck, getCacheStatsHandler, getDefaults, getMetrics (some), getAllUsers, getCategories, getSecurityTxt
- These ignore req — just call service and format response

**Controllers with `AuthenticatedRequest`** (overdue-penalty):
- Use `req.user!.id` — test must set user even though TS uses non-null assertion
