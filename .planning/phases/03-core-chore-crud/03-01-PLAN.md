  <name>Task 2: Create validator middleware and Zod schemas</name>

  <action>
    Follow the pattern from RESEARCH Pattern 3 (Zod Validation Middleware) exactly.

    **1. Create `backend-v2/src/middleware/validator.ts`:**
    - Export `validate(schema: ZodSchema)` as a middleware factory.
    - On success: `req.body = schema.parse(req.body)` then `next()`.
    - On ZodError: return `res.status(400).json({ success: false, error: { message: 'Validation failed', code: 'VALIDATION_ERROR', details: err.issues.map(e => ({ path: e.path.join('.'), message: e.message })) }, data: null })`.
    - On non-ZodError: `next(err)` to pass to errorHandler.
    - Import `ZodSchema, ZodError` from 'zod' and `Request, Response, NextFunction` from 'express'.

    **2. Create `backend-v2/src/schemas/template.schema.ts`:**
    - `createTemplateSchema`: `z.object({ title: z.string().min(1, 'Title is required').max(200), description: z.string().max(1000).optional(), points: z.number().int().min(1, 'Points must be at least 1'), category: z.string().min(1, 'Category is required').max(100) })` (D-04: free-text category, no enum).
    - `updateTemplateSchema`: `createTemplateSchema.partial()` — all fields optional for PATCH-like updates.

    **3. Create `backend-v2/src/schemas/assignment.schema.ts`:**
    - `createAssignmentSchema`: `z.object({ choreTemplateId: z.number().int().positive('Template ID is required'), assignedToId: z.number().int().positive('Assignee is required'), dueDate: z.string().refine(val => !isNaN(Date.parse(val)), 'Valid due date is required') })` (D-17: backend parameter names `choreTemplateId` and `assignedToId`, NOT `templateId`/`userId`).
    - `updateAssignmentSchema`: `z.object({ assignedToId: z.number().int().positive('Assignee is required').optional(), dueDate: z.string().refine(val => !isNaN(Date.parse(val)), 'Valid due date is required').optional() })` — only the two editable fields.

    Create the `schemas/` directory with `mkdir -p` if needed.
  </action>
