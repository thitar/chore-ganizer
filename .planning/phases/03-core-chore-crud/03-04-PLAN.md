<objective>
Create the frontend data layer: API clients using Axios with D-17 parameter mapping, and React Query hooks for server state management. This bridges the backend API (Plans 03-02, 03-03) to the UI pages (Plans 03-05, 03-06, 03-07).

Purpose: The API layer encapsulates all HTTP concerns (base URL, credentials, parameter mapping) so pages only deal with typed functions. The hooks provide loading/error/data states and automatic cache management.

Output: Six files — three API modules, three hook modules. Type-safe, follow existing patterns from auth.api.ts and useAuth.tsx.
</objective>

Backend API shapes (implemented by Plans 03-02, 03-03):
- GET /api/templates → `{ success: true, data: ChoreTemplate[] }`
- POST /api/templates body: `{ title, points, category, description? }` → `{ success: true, data: ChoreTemplate }`
- PUT /api/templates/:id body: partial template fields → `{ success: true, data: ChoreTemplate }`
- DELETE /api/templates/:id → `{ success: true, data: { deleted: true } }`
- GET /api/assignments → `{ success: true, data: Assignment[] }` (role-scoped)
- POST /api/assignments body: `{ choreTemplateId, assignedToId, dueDate }` → `{ success: true, data: Assignment }`
- PUT /api/assignments/:id body: `{ assignedToId?, dueDate? }` → `{ success: true, data: Assignment }`
- POST /api/assignments/:id/complete → `{ success: true, data: Assignment }`
- POST /api/assignments/:id/uncomplete → `{ success: true, data: Assignment }`
- DELETE /api/assignments/:id → `{ success: true, data: { deleted: true } }`

    **1. Create `frontend-v2/src/api/templates.api.ts`:**
    - Axios instance: `axios.create({ baseURL: '/api/templates', withCredentials: true })`.
    - Export `Template` interface: `{ id: number; title: string; description: string | null; points: number; category: string | null; createdById: number; createdAt: string; updatedAt: string }`.
    - Export `getAll(): Promise<Template[]>` — `api.get('/')` → `response.data.data`.
    - Export `create(data: { title: string; points: number; category: string; description?: string }): Promise<Template>` — `api.post('/', data)` → `response.data.data`.
    - Export `update(id: number, data: { title?: string; points?: number; category?: string; description?: string }): Promise<Template>` — `api.put(\`/${id}\`, data)` → `response.data.data`.
    - Export `delete_(id: number): Promise<{ deleted: true }>` — `api.delete(\`/${id}\`)` → `response.data.data`.
    - No parameter mapping needed for templates (same names on frontend/backend).

    **2. Create `frontend-v2/src/api/assignments.api.ts`:**
    - Axios instance: `axios.create({ baseURL: '/api/assignments', withCredentials: true })`.
    - Export `Assignment` interface: `{ id: number; choreTemplateId: number; assignedToId: number; dueDate: string; status: 'PENDING' | 'COMPLETED'; completedAt: string | null; pointsAwarded: number | null; notes: string | null; createdAt: string; template: { id: number; title: string; points: number; category: string | null }; assignedTo: { id: number; name: string; color: string } }`.
    - Export `getAll(): Promise<Assignment[]>` — `api.get('/')` → `response.data.data`. No query params (D-09: frontend-side filtering).
    - Export `create(data: { templateId: number; userId: number; dueDate: string }): Promise<Assignment>` — **IMPORTANT (D-17):** Map frontend names to backend names in the request body: `api.post('/', { choreTemplateId: data.templateId, assignedToId: data.userId, dueDate: data.dueDate })` → `response.data.data`.
    - Export `update(id: number, data: { userId?: number; dueDate?: string }): Promise<Assignment>` — Map `userId` → `assignedToId` in body: `api.put(\`/${id}\`, { ...(data.userId && { assignedToId: data.userId }), ...(data.dueDate && { dueDate: data.dueDate }) })` → `response.data.data`.
    - Export `complete(id: number): Promise<Assignment>` — `api.post(\`/${id}/complete\`)` → `response.data.data`.
    - Export `uncomplete(id: number): Promise<Assignment>` — `api.post(\`/${id}/uncomplete\`)` → `response.data.data`.
    - Export `delete_(id: number): Promise<{ deleted: true }>` — `api.delete(\`/${id}\`)` → `response.data.data`.

    Parameter mapping (D-17) happens ONLY in the `create` and `update` functions. `getAll`, `complete`, `uncomplete`, `delete_` have no mapping.

    **3. Create `frontend-v2/src/api/users.api.ts`:**
    - Axios instance: `axios.create({ baseURL: '/api/users', withCredentials: true })`.
    - Export `UserSummary` interface: `{ id: number; name: string; role: string; color: string }`.
    - Export `getAll(): Promise<UserSummary[]>` — `api.get('/')` → `response.data.data`.
    - No parameter mapping needed. No mutation functions (Phase 6 will add user management mutations).
    - This supports the CHORE-02 empty-state fix: the AssignmentsPage create form needs a user dropdown populated from `GET /api/users` (built in Plan 03-03).

  <name>Task 2: Create useTemplates.tsx, useAssignments.tsx, and useUsers.tsx hooks</name>

  <action>
    Follow useAuth.tsx pattern exactly (RESEARCH Pattern 5). Each hook uses `useQuery` for fetches and `useMutation` for mutations with `queryClient.invalidateQueries` on success.

    **1. Create `frontend-v2/src/hooks/useTemplates.tsx`:**
    - Export `useTemplates()` hook returning `{ templates, isLoading, error, createTemplate, isCreating, updateTemplate, isUpdating, deleteTemplate, isDeleting }`.
    - `useQuery({ queryKey: ['templates'], queryFn: templatesApi.getAll })` — `data: templates = []`, `isLoading`, `error`.
    - `createTemplate` wrapper: `const createTemplate = (data) => createMutation.mutateAsync(data)`. The underlying `useMutation({ mutationFn: templatesApi.create, onSuccess: () => queryClient.invalidateQueries({ queryKey: ['templates'] }) })`. Expose `isPending` as `isCreating`.
    - `updateTemplate` wrapper: `const updateTemplate = (id, data) => updateMutation.mutateAsync({ id, data })`. IMPORTANT: the wrapper takes `(id, data)` as two separate arguments (what pages call), but `mutateAsync` receives a single `{ id, data }` object (what `mutationFn` expects). Without the wrapper, pages calling `updateTemplate(id, data)` would only pass `id` as the first argument — `data` would be undefined.
    - Same pattern for `deleteTemplate`: `const deleteTemplate = (id) => deleteMutation.mutateAsync(id)`.
    - Import `useQueryClient` at hook top level.

    **2. Create `frontend-v2/src/hooks/useAssignments.tsx`:**
    - Export `useAssignments()` hook returning `{ assignments, isLoading, error, createAssignment, updateAssignment, completeAssignment, uncompleteAssignment, deleteAssignment, isCreating, isUpdating, isCompleting, isUncompleting, isDeleting }`.
    - `useQuery({ queryKey: ['assignments'], queryFn: assignmentsApi.getAll })` — `data: assignments = []`.
    - Each mutation follows the wrapper pattern: `const createAssignment = (data) => createMutation.mutateAsync(data)`, `const updateAssignment = (id, data) => updateMutation.mutateAsync({ id, data })`, `const completeAssignment = (id) => completeMutation.mutateAsync(id)`, etc.
    - Each with `onSuccess: () => queryClient.invalidateQueries({ queryKey: ['assignments'] })`.
    - Each exposes `isPending` with a descriptive name (e.g., `isCompleting`).

    **3. Create `frontend-v2/src/hooks/useUsers.tsx`:**
    - Export `useUsers()` hook returning `{ users, isLoading, error }`.
    - `useQuery({ queryKey: ['users'], queryFn: usersApi.getAll, staleTime: 5 * 60 * 1000 })` — users rarely change; 5-min stale time avoids redundant refetches.
    - No mutations needed — Phase 6 adds user management. This is strictly a read-only hook for dropdowns.
    - Returns `UserSummary[]` typed from `usersApi.getAll()`.

    Both hooks must NOT be wrapped in Context providers — they are standalone hooks, unlike useAuth which uses Context. Each consumer (page) calls the hook directly.
