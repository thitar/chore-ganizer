**Formatters** — Create `backend/src/services/notification.formatters.ts` with 3 named pure functions per D-02 and the agent's discretion (3 named functions, one per event):

Each function accepts a typed input and returns `{ title: string, body: string, priority: 1|2|3|4|5, tags: string[], click: string }`.

Interface definitions:
- `AssignmentInfo`: `{ id: number, template: { title: string, points: number }, dueDate: Date }`
- `UserInfo`: `{ name: string }` (accepted but NOT used in body — per NOTIFY-08 lock-screen privacy)

Functions:
1. `assignedBody(a: AssignmentInfo)` — title: `'Chore-Ganizer'`, body: `` `${a.template.title} — due ${a.dueDate.toISOString().slice(0, 10)}` ``, priority: `3`, tags: `['clipboard', 'bell']`, click: `` `/chores/${a.id}` `` (relative path per NOTIFY-08)
2. `dueSoonBody(a: AssignmentInfo)` — title: `'Chore-Ganizer'`, body: `` `${a.template.title} — ${a.template.points} pts, due today` ``, priority: `4`, tags: `['warning', 'alarm_clock']`, click: `` `/chores/${a.id}` ``
3. `completedBody(a: AssignmentInfo, _completer: UserInfo)` — title: `'Chore-Ganizer'`, body: `` `${a.template.title} — +${a.template.points} points earned` ``, priority: `2`, tags: `['white_check_mark', 'star']`, click: `` `/chores/${a.id}` ``

Key constraints (per NOTIFY-08): body must NEVER contain user name. The `_completer` parameter is prefixed with underscore to signal it's intentionally unused. Click path is relative (`/chores/{id}`), not absolute.
