**sendNtfy function** — the core transport, fire-and-forget with internal error swallowing:

```typescript
export async function sendNtfy(
  topic: string | null,
  title: string,
  body: string,
  opts: { priority?: 1|2|3|4|5; tags?: string[]; click?: string } = {}
): Promise<void>
```

Behavior:
1. Early return if `!isNtfyConfigured || !topic` — no logging, no side effects (per Pitfall 9)
2. Compose URL: `` `${getNtfyConfig().baseUrl}/${encodeURIComponent(topic)}` `` — `encodeURIComponent` handles special chars in topic
3. Build headers object: `Title` (string), `Priority` (string of number, default 3), `Tags` (comma-joined if array, omitted if empty), `Click` (if provided)
4. `await fetch(url, { method: 'POST', body, headers, signal: AbortSignal.timeout(3000) })` — 3-second timeout prevents hang on unreachable ntfy (per Pitfall 3)
5. Wrap entire fetch in try/catch — on error, `console.warn(`[ntfy] send failed for topic ${topic}: ${err instanceof Error ? err.message : String(err)}`)` and return. Never throw. Never let a rejection escape (per Pitfall 2).

**Domain wrappers** — 3 exported functions per D-01 and D-03 (accept pre-resolved Prisma objects):

Type definition (per D-03):
```typescript
type AssignmentWithIncludes = {
  id: number

}
```

1. `notifyChoreAssigned(assignment: AssignmentWithIncludes): void` — extract `topic = assignment.assignedTo.ntfyTopic`, early return if null, destructure formatter output from `assignedBody(assignment)`, fire `void sendNtfy(topic, title, body, { priority, tags, click })`

2. `notifyChoreDueSoon(assignment: AssignmentWithIncludes): void` — same pattern, uses `dueSoonBody(assignment)`

3. `notifyChoreCompleted(assignment: AssignmentWithIncludes, parents: { ntfyTopic: string | null }[]): void` — iterate `parents` array, skip entries where `ntfyTopic` is null, fire `void sendNtfy(topic, ...)` for each parent with a topic using `completedBody(assignment, { name: '' })`

- With empty parents array: does NOT call sendNtfy
- With all parents null topics: does NOT call sendNtfy
