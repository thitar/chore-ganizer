<objective>
Modify `assignmentService.getAll()` to call `generateOccurrences()` lazily before querying, then fetch both regular assignments and recurring occurrences, merge them into a single sorted array with a `type` discriminator. This makes occurrences visible on existing MyChoresPage, AssignmentsPage, and DashboardPage with zero frontend changes.

Purpose: RECUR-02 requires occurrences to appear on demand. Piggybacking on the existing assignment endpoint with a type field is the simplest integration — existing frontend pages render the data without changes.
</objective>

  <action>
    Modify `getAll(userId, role)` in assignment.service.ts:

    1. **Import** `generateOccurrences` from `../services/recurring.service`.

    2. **Call lazy generation** at the start:
       ```typescript
       const now = new Date()
       const from = new Date(now.getFullYear(), now.getMonth(), 1)
       const to = new Date(now.getFullYear(), now.getMonth() + 1, 0)
       await generateOccurrences(from, to)
       ```
       This generates missing occurrences for the current month. A full-range option isn't needed — the frontend filter defaults to current month per D-11.

    3. **Fetch occurrences alongside assignments:**
       ```typescript
       const occWhere = role === 'PARENT' ? {} : { assignedToId: userId }
       const [assignments, occurrences] = await Promise.all([
         prisma.choreAssignment.findMany({ where, include: {...}, orderBy: { dueDate: 'asc' } }),
         prisma.recurringOccurrence.findMany({
           where: occWhere,
           include: {
             chore: { include: { template: { select: { id: true, title: true, points: true, category: true } } } },
             assignedTo: { select: { id: true, name: true, color: true } },
           },
           orderBy: { dueDate: 'asc' },
         }),
       ])
       ```

    4. **Map occurrences to match assignment shape:**
       ```typescript
       const mapped = occurrences.map(occ => ({
         id: occ.id,
         type: 'RECURRING',
         choreTemplateId: occ.chore.choreTemplateId,
         assignedToId: occ.assignedToId,
         dueDate: occ.dueDate.toISOString().split('T')[0],
         status: occ.status,
         completedAt: occ.completedAt?.toISOString() ?? null,
         pointsAwarded: occ.pointsAwarded,
         notes: null,
         createdAt: occ.createdAt.toISOString(),
         template: occ.chore.template,
         assignedTo: occ.assignedTo,
       }))
       ```

    5. **Merge and sort:**
       ```typescript
       const merged = [
         ...assignments.map(a => ({ ...a, type: 'REGULAR', dueDate: a.dueDate.toISOString().split('T')[0] })),
         ...mapped,
       ].sort((a, b) => a.dueDate.localeCompare(b.dueDate))
       return merged
       ```

    **Note:** The `dueDate` field in ChoreAssignment is stored as DateTime (ISO string in JSON). The occurrences also return DateTime. Normalize both to `YYYY-MM-DD` strings for consistent frontend comparison. The existing frontend already handles `dueDate` as a string in filters (`a.dueDate < dateFrom`).
