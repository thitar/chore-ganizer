## Milestone Summary

### Key Decisions

- Fatal startup guards required for all security-critical environment variables
- Production error responses sanitize 5xx messages while preserving 4xx user-facing messages
- Full error details (including stack traces) remain in server-side logs
- Exported ApiClient class for testability with per-test instantiation
- `_csrfRetryCount` on config instead of WeakMap for simplicity
- `Math.round` for integer penalty calculation
- `getStartOfTodayUTC()` helper to eliminate duplicated date boundary logic
- Double-penalty guard returns HTTP 409 ALREADY_PENALIZED
- ESLint v10 with flat config format (eslint.config.cjs)
- Winston structured logging uses (message, metadata) signature
- Controller split: swagger JSDoc in route files to stay under 300-line limit
- Mounted occurrences router before CRUD router to prevent route shadowing
- Barrel re-exports for backward compatibility during controller splits
- Option B (keep String with Zod validation) for recurrence rule JSON storage

### Issues Resolved

- SESSION_SECRET no longer has development fallback — app refuses to start without it
- CSRF retry loop vulnerability mitigated (capped at 1 retry per request)
- Child users now see "Access Denied" toast before silent redirect
- Backend/frontend version drift now caught by CI gate
- 40+ console statements migrated to Winston structured logging
- 966-line monolithic controller split into focused 299-line and 288-line controllers
- Overdue penalties hardened: double-penalty guard, integer math, 18 edge case tests
- Frontend error handling (401, network failure, 500) covered by 6 new tests

### Issues Deferred

- API Rate Limiting Visibility — admin UI feature request (MISSING_FEATURES.md)
- `recurrence.service.ts` edge case tests — leap years, month-end dates, DST transitions

### Technical Debt Incurred

- SQLite scaling limits for multi-family use (documented risk)
- In-memory caching per-instance limitation (documented risk)
- Integration test DB lifecycle (fragile area, documented)
- Auth state with React StrictMode (fragile area, documented)
- Prisma / Lucide dependency risks (tracked)
